# Claude Context — pointer only

Start with `AGENT_CONTEXT.md`. Everything that applies to every agent
applies to Claude Code.

## Claude-specific entry points

- Adapter surface. Canonical lives in `.windsurf/`; Claude Code
  consumes via its own rule / command files.
- Rules: `.claude/rules/` (mapped one-to-one from Windsurf)
- Commands: `.claude/commands/` (slash-commands mirroring workflows)
- Skills index: `.claude/skills/INDEX.md`
- MCP config: `~/.claude.json` or repo-local `.claude/mcp.json`
  (repo-local version is the `*.example.json` only).

## Claude-specific behaviour

- Use `AGENTS.md` as the authority for protocol and invariants.
- The `agentic_manifest.yaml` declares which rules Claude consumes;
  CI drift-checks against it.
- Do not install or configure MCPs without explicit human approval.
- Treat production writes as STOP.
- Before large edits, consult `scripts/validate_agentic_manifest.py`
  and run it with `--strict` after any rule / command edit.
