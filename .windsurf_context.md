# Windsurf Context — pointer only

Start with `AGENT_CONTEXT.md`. Everything that applies to every agent
applies to Windsurf.

## Windsurf-specific entry points

- Authoritative surface of this template. Other IDE adapters derive
  from these files (ADR-023).
- Rules: `.windsurf/rules/` (15 files)
- Skills: `.windsurf/skills/` (16 folders, each with `SKILL.md`)
- Workflows: `.windsurf/workflows/` (12 files)
- MCP config: consumer responsibility, see
  `~/.codeium/windsurf/mcp_config.json` (never committed).

## Windsurf-specific behaviour

- Use `AGENTS.md` as the authority for protocol and invariants.
- Consult `templates/config/agentic_manifest.yaml` before editing any
  rule / skill / workflow — its entry must match reality on disk.
- Do not install or configure MCPs without explicit human approval.
- Treat production writes as STOP.
- For code changes, run the narrow verifier first:
  `python3 scripts/validate_agentic.py --strict` and
  `python3 scripts/validate_agentic_manifest.py --strict`.
