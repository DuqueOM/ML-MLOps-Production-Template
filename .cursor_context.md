# Cursor Context — pointer only

Start with `AGENT_CONTEXT.md`. Everything that applies to every agent
applies to Cursor.

## Cursor-specific entry points

- Adapter surface. Canonical lives in `.windsurf/`; Cursor consumes
  via its own rule / command files.
- Rules: `.cursor/rules/` (14 files, mapped one-to-one from Windsurf)
- Commands: `.cursor/commands/` (slash-commands mirroring workflows)
- Skills index: `.cursor/skills/INDEX.md`
- MCP config: `~/.cursor/mcp.json` or repo-local `.cursor/mcp.json`
  (repo-local version is the `*.example.json` only — the live config
  with real endpoints never enters the repo).

## Cursor-specific behaviour

- Use `AGENTS.md` as the authority for protocol and invariants.
- The `agentic_manifest.yaml` declares which rules Cursor consumes;
  CI drift-checks against it.
- Do not install or configure MCPs without explicit human approval.
- Treat production writes as STOP.
- Run `python3 scripts/validate_agentic_manifest.py --strict` after
  editing any Cursor rule or command file.
