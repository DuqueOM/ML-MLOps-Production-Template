# Codex Context — pointer only

Start with `AGENT_CONTEXT.md`. Everything that applies to every agent
applies to Codex.

## Codex-specific entry points

- Adapter surface (Sprint-4 F5). Canonical lives in `.windsurf/`;
  Codex consumes via thin pointers + `AGENTS.md`.
- Skills (4 in Sprint-4): `.codex/skills/*.md` — pointer files
  referencing `.windsurf/skills/<id>/SKILL.md`.
- Automations (3): `.codex/automations/*.md` — Codex-specific
  cron/event mappings.
- MCP config: `.codex/mcp.json` (gitignored, adopter-specific).
  Reference template: `.codex/mcp.example.json`.

## Codex-specific behaviour

- Use `AGENTS.md` as the authority for protocol and invariants.
- Codex never duplicates skill bodies — every `.codex/skills/*.md`
  must reference its canonical Windsurf source (contract test
  enforces this in CI).
- Do NOT install or configure MCPs without explicit human approval
  (ADR-023 I-3). The Codex MCP install mode is `user_confirmed`.
- Treat production writes as STOP. Codex never runs `terraform apply`
  or `kubectl apply` against staging or production.
- Run `python3 scripts/validate_agentic_manifest.py --strict` and
  `make mcp-check` before pushing changes that touch any rule,
  skill, workflow, or MCP entry.
