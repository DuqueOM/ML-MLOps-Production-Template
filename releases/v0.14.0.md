# v0.14.0 — Enterprise Adoption Remediation

Date: 2026-05-03

## Release Classification

Pre-GA hardening release on the active `v0.x` line. This release improves
first-adopter enterprise readiness, but it does not claim L4 production
verification. `v1.0.0` remains reserved for real cloud E2E evidence across
GKE and EKS.

## Why This Release Exists

The Staff-level audit found that the repo was strong as an enterprise
portfolio artifact but still had practical adoption gaps in the first
scaffold, first CI run, and first non-agentic operations review.

## Highlights

- Scaffolded CI now runs from the service repo root instead of expecting a
  nested `${ServiceName}/` directory.
- GCP/AWS deploy workflows now build root-context images named
  `{service-name}-predictor`, matching Kustomize overlays and digest
  pinning.
- `pyproject.toml` now discovers the service package under `src/` and
  aligns dependency/tooling versions with the generated requirements and
  pre-commit chain.
- Inference now applies the training `FeatureEngineer.transform_inference()`
  path before prediction, with an explicit opt-out only for documented
  serialized-pipeline exceptions.
- Previously skipped train/inference parity coverage is now an executable
  scaffolded test.
- Non-agentic runbook references now resolve to real Day-2 procedures.
- README records the post-remediation score delta:
  - Overall: 8.1 -> 8.7/10
  - Staff MLOps portfolio: 9.0 -> 9.3/10
  - Immediate enterprise adoption: 7.4 -> 8.4/10

## Validation

Expected local validation set:

```bash
python3 scripts/ci_verify_yaml.py
python3 scripts/validate_agentic.py --strict
python3 scripts/validate_agentic_manifest.py --strict
python3 scripts/ci_verify_workflows.py
python3 scripts/ci_verify_targeted.py
bash scripts/test_scaffold.sh
SCAFFOLD_SMOKE=1 bash scripts/test_scaffold.sh
```

The smoke command is allowed to be deferred locally when dependency
installation or network time makes it unsuitable for the workstation; the
status must be recorded in `VALIDATION_LOG.md`.
