## Highlights

### One-command bootstrap

- `scripts/bootstrap.sh` — OS-aware installer (Linux / macOS / WSL).
  Verifies Python 3.11+, Docker, kubectl, terraform, git, make;
  installs Python deps; configures MCPs interactively; installs
  pre-commit hooks; runs the minimal example end-to-end
- Flags: `--skip-mcp`, `--skip-demo`, `--check-only`
- Helpers: `scripts/_lib/detect_os.sh`, `install_deps.sh`,
  `configure_mcp.sh`
- Makefile: `make bootstrap`, `make bootstrap-check`

### Agentic system validator

- `scripts/validate_agentic.py` — validates `.windsurf/` structure:
  * Rule frontmatter (`trigger`, `description`, `globs`)
  * Glob patterns match real files (catches dead rules)
  * Skill `SKILL.md` contracts (`name`, `description`, `allowed-tools`)
  * Workflow frontmatter
  * AGENTS.md cross-references (no orphan skills/workflows)
- CI job `agentic-system` in `validate-templates.yml`
- Make target `validate-agentic` chained into `validate-templates`

### Governance module (opt-in)

- `templates/governance/README.md` — when/how to enable approval gates
- `ROLES.md` — ML Engineer / Tech Lead / Platform Engineer responsibilities
- `github-environments.yml` — reference config (staging + prod with
  `required_reviewers` and 24h soak)
- `promote-with-approval.yml` — GitHub Actions promotion workflow with
  MLflow stage transitions and audit tags
- `promote_to_stage.sh` — CLI for MLflow Registry transitions
- **ADR-002** — Model Promotion Governance: why opt-in, why GitHub
  Environments + MLflow stages

### Scaffolder end-to-end test

- `scripts/test_scaffold.sh` — runs `new-service.sh` in an isolated
  temp dir and validates:
  * Zero remaining `{ServiceName}`/`{service}`/`{SERVICE}` placeholders
  * Critical files + directories present
  * `src/{service}/` renamed correctly to `src/<slug>/`
  * Generated Python files parse
  * Both Kustomize overlays render (GCP + AWS)
  * `pytest` can collect scaffolded tests
- CI job `scaffold-e2e`
- Make target `test-scaffold`

### Feast integration pattern

- **ADR-003** — documents integration without core template changes:
  external feature repo, service as Feast client, Pandera preserved,
  4-phase migration checklist
