# v0.15.2 - FastAPI Template Contract Hardening

Release date: 2026-05-06

This patch release hardens the existing FastAPI service scaffold. It
does not introduce a new framework or a second API template.

## What changed

- Added `docs/FASTAPI_TEMPLATE_CONTRACT.md`, a concise reviewer-facing
  contract for required endpoints, readiness, auth, feature parity,
  metrics, prediction logging, CORS, error envelope, and modelless
  startup restrictions.
- Added `templates/service/tests/test_fastapi_template_contract.py` to
  freeze the structural serving invariants in every scaffolded service.
- Updated the canonical Python serving rule and related skills/workflows
  so agents describe the current `app/main.py` + `app/fastapi_app.py`
  split accurately.
- Updated cloud deploy and load-test smoke examples to call `/ready` and
  use a schema-valid scaffold payload.
- Updated `scripts/test_scaffold.sh` so the full smoke chain runs the
  new FastAPI contract test against a freshly generated service.
- Updated README and Quick Start onboarding to make the included FastAPI
  service surface explicit.
- Normalized the `v0.14.0` release-note follow-ons heading to the
  canonical release-note contract.

## Known limitations

- This release does not claim L4 production rollout evidence.
- Real cloud GKE/EKS validation remains adopter-owned evidence.
- OpenTelemetry under load remains adopter-side because tracing is
  opt-in and depends on a real collector.
