# v0.13.0 — Pre-GA Enterprise Hardening

Date: 2026-05-03

## Release Classification

`v0.13.0` starts the public pre-GA hardening channel. Historical
`v1.0.0` through `v1.12.0` tags are preserved as immutable audit
snapshots, but `v1.0.0` is now reserved for the first release with real
cloud golden-path evidence across GKE and EKS.

## Why This Release Exists

This release closes two public-signal problems:

1. The repo had enterprise-grade intent but still tolerated false
   positives in scaffold smoke and golden-path validation.
2. The agentic layer was drifting toward per-IDE duplication: Windsurf
   held the full source, while Cursor, Claude, and Codex had uneven
   coverage.

## Highlights

- Full agentic adapter parity:
  - 15 canonical rules.
  - 16 canonical skills.
  - 12 canonical workflows.
  - Windsurf as canonical body store.
  - Cursor, Claude, and Codex as generated pointer adapters.
- New `scripts/sync_agentic_adapters.py` renderer.
- Strict adapter validation in `validate_agentic_manifest.py`.
- Codex now has `.codex/{rules,skills,workflows}/` pointers, not only
  four Sprint-4 skill files.
- Golden-path hardening:
  - scaffold smoke fails on skipped install/test/snapshot paths;
  - kind deploy requires real health, readiness, predict, and metrics
    checks through a CI-only synthetic model path.
- Quality-gate hardening:
  - `require_eda_artifacts` can block training/promotion when the EDA
    artifact chain is missing.
- Infrastructure hardening:
  - AWS lock-table naming aligned by environment;
  - GCP private endpoint default tightened;
  - GCP node OAuth scopes reduced;
  - runtime identities scoped closer to artifacts/buckets.

## MCP Installation Notes

For this workstation, the Codex MCP entries were added to
`~/.codex/config.toml`, following the current Codex MCP configuration
format. Installed local npm-based servers:

- `kubectl-mcp-server@1.24.0`
- `mcp-prometheus@2.1.1`

Configured but not fully runnable on this workstation:

- `hashicorp/terraform-mcp-server:latest` via Docker. Docker is present,
  but the daemon was not running during validation, so the image could
  not be pulled.

No credentials were written to the repo or to Codex config. GitHub and
Prometheus MCPs expect environment variables.

## Validation

Required local validation before tagging:

```bash
conda run -n ml python scripts/sync_agentic_adapters.py --check
conda run -n ml python scripts/validate_agentic_manifest.py --strict
conda run -n ml python scripts/mcp_doctor.py --mode check
conda run -n ml python scripts/ci_verify_yaml.py
conda run -n ml python scripts/ci_verify_workflows.py
conda run -n ml python scripts/ci_verify_targeted.py
conda run -n ml pytest templates/service/tests/test_agentic_manifest_contract.py templates/service/tests/test_codex_adapter_contract.py templates/service/tests/test_mcp_registry_contract.py -q
```

## Known follow-ons

- Real GKE/EKS cloud golden-path execution remains the gate for future
  `v1.0.0`.
- Terraform MCP requires a running Docker daemon or an alternate
  non-Docker server install path.
- Admission-level signature verification in kind remains a follow-on;
  current verification combines workflow-level `cosign verify` with
  digest enforcement in the test cluster.
